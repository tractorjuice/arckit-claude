# Bulk-Build ArcKit Artefacts

You are invoking the **ArcKit build harness** — the `arckit-build` skill orchestrates parallel `/arckit:*` artefact generation against a YAML recipe.

This command is a thin wrapper. The orchestration logic lives in `${CLAUDE_PLUGIN_ROOT}/skills/arckit-build/SKILL.md`. **Invoke that skill now via the Skill tool**, passing the user's arguments verbatim:

## When to use this command

When you have several `/arckit:*` artefacts to produce for a single project and don't want to run them sequentially in main-context (which exhausts context past ~5 artefacts). Typical use:

- Fresh project: `/arckit:build 001` builds the full UK-SaaS recipe in waves
- Resume after failure: `/arckit:build 001 --resume`
- Dry run: `/arckit:build 001 --plan` (recommended first step on any new project)
- Custom recipe: `/arckit:build 002 --recipe uk-mod-sovereign`

## What it does (overview only — full detail lives in the skill)

1. Resolves the recipe (default `uk-saas`; precedence: `.arckit/recipes/{NAME}.yaml` > `${CLAUDE_PLUGIN_ROOT}/skills/arckit-build/recipes/{NAME}.yaml`)
2. Computes the dependency DAG and groups targets into parallel waves
3. Smoke-tests subagent skill access (~5 sec halt-or-go check)
4. For each wave: dispatches one `general-purpose` subagent per target in a single parallel message; collects summaries; validates outputs; commits the wave as one atomic git commit
5. Updates `projects/{P}/.arckit/state.json` after every wave for resumability
6. Halts on failure (state preserved; `--resume` picks up cleanly)
7. Runs post-build hooks (`/arckit:health`, `/arckit:pages`)

Invoke the skill now and let it drive. The skill is responsible for argument parsing, recipe loading, wave dispatch, and reporting — this command does nothing else.
